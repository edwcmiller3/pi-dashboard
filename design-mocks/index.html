pi-dashboard — design mock gallery

The chosen direction (instrument HUD, variant D) with its accepted state references, plus the three unchosen design directions kept for the record. Same information architecture and fixture content throughout (see CONTENT.md), each a self-contained HTML file at the 1280×800 kiosk viewport. Click a title to open the live mock. Local-only — nothing here is wired to the app. (Superseded HUD intermediates and rejected state mocks were deleted 2026-07-12.)

Paper editorial — “the daily broadsheet”

E-ink/print aesthetic: warm paper, ink-only palette with one oxblood accent, Playfair/Newsreader, newspaper furniture (masthead clock, column rules, almanac table).

Neo-brutalist

Flat saturated panels, 4px black borders, hard offset shadows, Archivo Black numerals, inverted clock slab, coral “NOW” bar, black ticker footer.

Swiss / mono exposed grid

Engineering-sheet modular grid with indexed cells (01/TIME … 08/STATUS), JetBrains Mono metadata + huge grotesque numerals, single surgical Swiss-red accent.

HUD variant D — THE KEEPER (everything agreed to date)

Green-phosphor instrument cluster: 240° temp dial on a computed data-driven scale window (60°F span snapped to 10s — here 60–120), linear solar day-tape, channel-row forecast on the shared scale, edge-block ACTIVE row, flavor text purged, units declared once in the header, right-aligned status footer with RESYNC.

HUD D state — winter (data-driven scale)

Critique should-fix 1: the 50–110°F scale becomes a computed 60° window snapped to 10s (here 10–70 for a 28° January day); dial, ruler, and range bars all derive from it. Adds snow/overcast glyphs and amber crystal precip pills.

HUD D state — light calendar week

Critique should-fix 4: zero-event days render an explicit — NO ENTRIES — line on the event grid, the horizon extends to seven days, and the manifest closes with an END OF SCHEDULE tick rule so emptiness reads intentional.

Warm Scandinavian light mode
